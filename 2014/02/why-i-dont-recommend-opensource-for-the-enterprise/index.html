Why I don’t recommend opensource for the enterprise

February 2, 2014 trajano Leave a comment

My view is an enterprise needs to be sustainable and your architecture is only as sustainable as the least supported component.  My view for delivery is different, but that’s another story.

I was recently pondering this when I was working on my logo which depended on batik-maven-plugin which hasn’t been updated since 2009.  Of course I took it as a calculated risk, especially since I can code my own batik-maven-plugin.  Imagining this if it were an enterprise that relied on an open source component that suddently didn’t get any more support.

Open source does have support partners, but can it handle your company?  Technology tends to come and go like fashion trends and when interest wanes, the support wanes or becomes more expensive (just like the cost of extended support for XP).

The good part is  at least two large vendors: IBM and Oracle provide support for their products that are based on open source software so at least you’re going to be better covered.

If someone asks me whether I would use MySQL or Ruby on Rails, and Git vs DB2, WebSphere and Rational Team Concert, I would go for the later set.

  1. There’s a single BIG vendor that owns the product that simplifies the support costs.  Though you’re reliant on them so it better be a big vendor that you won’t worry about going belly up anytime soon.
  2. Having a vendor to support you allows you to pass the onus of fixing bugs to the vendor rather than getting and keeping the skills in-house which would cost more in the future or you’d get someone stuck that doesn’t grow anyway.
