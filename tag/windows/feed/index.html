I use yajsw because it is free unlike JSW which requires you to pay to have a service running on a 64-bit machine.  Another app I use is Process Explorer to determine the PID of the app.

I start out with making ElasticSearch as service.  To do this I simply run elasticsearch.bat and locate the app on Process Explorer.

I create a YAJSW folder for every service I am going to make, in my case C:\opt\yajsw-elasticsearch for elasticsearch and c:\opt\yajsw-logstash for logstash.  Not sure if it is possible to avoid this, but this was the most expedient way to solve my issue.

Once I find the app, I note the PID and go to the command line where I put in the YAJSW files.

cd c:\opt\yajsw\bat
genConfig.bat <pid>

I make few changes to the configuration

wrapper.app.env.HOME=C:/opt/logstash/sincedb
wrapper.tray = false
wrapper.console.title=elasticsearch
wrapper.ntservice.name=elasticsearch
wrapper.ntservice.displayname=elasticsearch
wrapper.ntservice.description=elasticsearch

Then install the service.

installService.bat

I then start Service Manager and locate org.elasticsearch.bootstrap.ElasticSearch and start it.

I then test after a minute by going to http://localhost:9200/

For logstash, I do the same process except I have to make a few other changes to the wrapper.conf file namely

wrapper.app.env.HOME=C:/opt/logstash/sincedb
wrapper.java.maxmemory=128
wrapper.tray = false
wrapper.console.title=logstash
wrapper.ntservice.name=logstash
wrapper.ntservice.displayname=logstash
wrapper.ntservice.description=logstash
wrapper.ntservice.dependency.1=elasticsearch
