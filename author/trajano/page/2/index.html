Got the Hangout update finally

November 16, 2013 trajano Leave a comment

These rolling updates are quite annoying. However, I finally got the fabled Hangout update that supports SMS.

My only other comparison is the MightyText app and iMessage.

Anyway the SMS addition to hang out is a welcome change. However it still lacks a few usability features.

First of all, you simply just have another SMS client, the only thing is you can see chat like messages all in one spot.

However, they are still treated like separate contacts rather than a unified contact view.

Try it out when you SMS a person who is on your hangout you do not see message logs that are unified, it is a different message log per method of contact.

It also does not sync up the messages with Hangout on the tablet. Unlike the MightyText app which does that job.

For now, I have uninstalled MightyText to see what else I could be missing.

In the iOS world

I recently got my first iPhone and I verified that iMessage can send SMS and chats at the same time (or at least give the illusion of it), but only on the Mac and the iPhone. There is no PC version.

Still, Hangouts a good start but also very disappointing.

Don’t use hosting24

August 20, 2013 trajano Leave a comment

I am in the process of moving away from 000webhost whose parent company is hosting24. Though I expected to do it later than this soon.

Seems like they hold your data hostage, as I expected, and I did put a bit of contingencies at least with most of the content, but the database was a bit of an issue because there was no direct access to the database to do periodic backups and I wasn’t able to put enough time to manage the staged deployment of content to a WordPress system hosted on 000webhost.

Anyway, for those who are cheap I guess 000webhost is still an okay choice provided you don’t use any facility aside from HTML hosting like for Maven sites.  However, I won’t recommend them for anything of use.

I won’t recommend their parent company, hosting24, just because they support such an unscrupulous practice to lure people in.  Instead I would likely refer people to either GoDaddy which already does the DNS or Dave’s Network which my friend uses for hosting in Canada at a relatively cheap rate.

In fact, I may switch to Dave’s Network if my website requires a bit more uptime to drive business, but as this website is still primarily a hobby for me I can’t justify paying more than $0.

For now it is going to take a couple of weeks before I totally restore the web site again.  However, this time I have better control over how things work.

Introduction to Cloud Computing

May 13, 2013 trajano Leave a comment

If you are reading this, you are using electricity at one point when your device is plugged into the wall. In a sense you are using a utility, the electric company, that provides a resource for you, electricity, remotely. Behind that wall socket is a vastly complex infrastructure that deals with providing and billing you for the electricity.

So how does that relate to cloud computing? Quite a bit.

Item Electric company Cloud computing
What resource do we consume Electricity Information, storage, processing power
How to get the resource remotely Wall socket Web browser or app
Where does the resource come from Coal, Nuclear, Solar, Wind, etc. People, data, hardware
What complex infrastructure is used to process the resources for our consumption? Power generators and transformers Computers, networking and software

Cloud computing provides consumable computing resources which are information, storage or processing power. It is important to not that it has to be consumable by a consumer otherwise it does not provide any useful benefit. It also needs to be a computing resource, network bandwidth and electricity are not computing resources because they are not computed, but they are needed just like wires are for electricity.

The consumer must be able to obtain the resource remotely and with ease. Just like a wall socket which hides all the wiring and conversions from coal, nuclear, solar, wind, etc. a web browser or app hides all the underlying complexity of the servers and networks including security and billing concerns. Although it may seem second nature for technical people, these systems use a complex infrastructure.

So where does cloud come from?

Laziness. When technical folk do network diagrams, we usually draw the Internet as a cloud because it is a vastly complex infrastructure and generally we don’t care about it the details, only in its existence. Except for the network diagrams it’s more of a network connection to the outside world which isn’t really a consumable computing resource much like electricity is not a consumable computing resource.

So what’s the advantage of cloud computing?

The main advantage of cloud computing is economies of scale. By simplifying access to the consumable computing resources, cloud providers can expand their market to make up the cost of producing the resources gets reduced the larger the consuming market is. As such it can be used to support the sustainability and growth goals for the provider.

The provider traditionally is outside the organization, just like the electric company to most organizations unless you’re the electric company. This brings us to the second advantage, predictable costs for service. Since most providers would provide a service level agreement for a specific payment, it makes things easier to budget for organizations to use cloud services.

Generally simpler access to the resources than dealing with the complexity of doing it in-house, this is how the providers remain competitive.

What’s the disadvantage?

Loss of control of the computing resources. For some organizations, privacy requirements would remove the case for using public cloud services. However, it does not impede on developing a private cloud within the organization in order to control the information.

An internal cloud simplifies IT project development within the organization as the IT infrastructure complexity for consumable computing resources can be simplified to an internal cloud, reducing the need for dealing with low level computing resources by IT projects and instead focus on the business needs.

In short…

Cloud computing is a complex infrastructure that provides consumable computing resources remotely and with ease.
